Starting from characterization (Phases 1–2) through research contribution development (Phase 3) and advanced extensions (Phases 4–5), the work culminated in the Elastic-Precision Expert Gating (EPEG) co-design framework — a publication-ready contribution targeting MLSys / ASPLOS / OSDI venues.

Phase 1: Scheduling-Aware Batching (SAB) — Characterization

Objective: Evaluate whether affinity-aware batching (grouping requests by expert routing similarity) can reduce All-to-All communication volume and improve serving throughput.

Models tested: Qwen3-235B, DeepSeek-R1, Llama4-Maverick Variants tested: FIFO baseline, SAB, SAB-AAE, SAB-Cooldown, SAB-Threshold, SAB-Hybrid

Key Results

Model Best SAB Variant Speedup vs. FIFO
Qwen3-235B SAB-Cooldown 1.04x
DeepSeek-R1 SAB-Hybrid 1.01x
Llama4-Maverick SAB-Threshold 1.02x

Key Finding

SAB provides marginal improvements (1–4%) because All-to-All communication is not the dominant bottleneck on fast NVLink. The scheduling overhead can sometimes negate the communication savings.

Phase 2: Distributed Interconnect-Aware Routing & HDFG — Negative Results

Objective: Investigate two standard distributed co-design hypotheses: (1) gate pruning (λc) to force local execution, and (2) Hierarchical Dispatch-Fetch Gating (HDFG) to stream expert weights instead of dispatching tokens.

Phase 3: Elastic-Precision Expert Gating (EPEG) — The Research Breakthrough

Objective: Dynamically assign expert precision (BF16 / FP8 / FP4) per token-expert route based on routing confidence, reducing both All-to-All network volume and expert compute latency.

Core Mechanism

For Top-8 routing (Qwen3-235B, DeepSeek-R1): - Top-2 experts (highest confidence): BF16 (scale = 1.0) - Middle-4 experts: FP8 (scale = 0.5) - Bottom-2 experts (tail): FP4 (scale = 0.25)

Average precision scale: 0.5625 → 43.75% network volume reduction + 43.75% compute acceleration

Phase 4: Dynamic Closed-Loop SLA Gating (EPEG-SLA)

Objective: Address the limitation that Static EPEG always sacrifices accuracy. Implement a dynamic feedback controller that activates quantization only when the system is congested.

Mechanism

  • Normal state (queue delay < 50ms): Thresholds $(0.0, 0.0)$ → pure BF16 → 0% accuracy loss
  • Congested state (queue delay ≥ 50ms): Thresholds $(0.50, 0.10)$ → aggressive quantization → SLA protection

Results

Load Variant Latency Queue Delay GSM8K Loss
Low EPEG-SLA 51.177s 0.05 ms 0.0000% ✓
Low Static EPEG 50.934s 0.04 ms 0.0810% ✗
High EPEG-SLA 3.588s 130.17 ms 0.1707%
High Static EPEG 2.201s 85.75 ms 0.0810%

Design Trade-off: EPEG-SLA achieves perfect accuracy under normal operations (addressing the reviewer concern about always-on quantization). Under bursts, it uses more aggressive thresholds than Static EPEG to rapidly drain queue backlogs, which temporarily increases accuracy cost but protects SLA targets.

Phase 5: Advanced Co-Design Extensions (CAPS & EPEG-Slice)

Objective: Address scheduling bubbles from heterogeneous precision loads (CAPS) and physical memory bandwidth overhead from format casting (EPEG-Slice).

CAPS (Confidence-Aware Priority Scheduling)

Reorders waiting requests using a sliding window ($N=10$) to minimize EP rank load variance, balancing BF16-heavy and FP4-heavy sequences within each batch.

EPEG-Slice (Hardware-Aware Bit-Plane Slicing)

Stores expert weights in decomposed bit-planes. The memory controller streams only the required planes (16 for BF16, 8 for FP8, 4 for FP4), achieving linear DRAM bandwidth savings without format casting kernels.

Publication Positioning

Venue Targets: MLSys, ASPLOS, OSDI, ISCA

Core Claim: EPEG demonstrates that dynamically quantizing expert routing based on per-token gating confidence breaks the MoE communication and compute tax, achieving 1.37–1.79x serving speedup with projected accuracy degradation under 0.1% across Qwen3-235B and DeepSeek-R1.

Differentiators vs. SOTA: - Pareto-dominates Uniform FP8 (DeepEP) on accuracy-speedup frontier - Safe no-op on Top-1 architectures (unlike blanket quantization) - Closed-loop SLA control (EPEG-SLA) guarantees 0% loss under normal loads - CAPS resolves scheduling bubbles from precision heterogeneity
